I. Commit

Repo & Fișiere de bază

01
Creare repository „Evaluare Nr2" pe GitHub
GitHub → New repository → Name: Evaluare-Nr2 → Create repository
02
Adăugare fișier index.ev.2.html (remote)
GitHub → Add file → Create new file → index.ev.2.html → Commit changes
03
Clonare repository local
git clone https://github.com/USER/Evaluare-Nr2.git  &  cd Evaluare-Nr2
04
Adăugare locală fișier settings.xml
touch settings.xml  →  git add settings.xml
05
Commit settings.xml
git commit -m "Commit file settings.xml"
06
Redenumire settings.xml → configuration.xml
git mv settings.xml configuration.xml
07
Commit configuration.xml
git commit -m "Commit configuration.xml"
08
Modificare mesaj ultimul commit (amend)
git commit --amend -m "Commit file configuration.xml"
09
Push toate modificările
git push origin main
10
Evaluare rezultat — I. Commit
Verificați pe GitHub că apar commiturile cu mesajele corecte
II. Commit

Reset & Push Force

01
Adăugare remotă page1.html
GitHub → Add file → page1.html → Commit changes
02
Actualizare repository local
git pull origin main
03
Revenire la „Commit file configuration.xml"
git log --oneline  →  git reset --hard <hash>
04
Actualizare repository remote (force)
git push origin main --force
05
Evaluare rezultat — II. Commit
page1.html dispare; istoricul se oprește la „Commit file configuration.xml"
I. Branch

Branch stage din main

01
Creare remote branch stage din main
GitHub → Branches → New branch: stage from main
02
Creare hosts.xml în branch stage + commit
GitHub (branch: stage) → Add file → hosts.xml → "Commit file hosts.xml"
03
Actualizare repository local
git fetch origin  →  git checkout stage  →  git pull origin stage
04
Merge stage → main (local + remote)
git checkout main  →  git merge stage  →  git push origin main
05
Evaluare rezultat — I. Branch
hosts.xml apare în main pe GitHub
II. Branch

Branch dev din stage

01
Creare local branch dev din stage
git checkout stage  →  git checkout -b dev
02
Creare fișier env.txt
touch env.txt  →  git add env.txt
03
Commit env.txt
git commit -m "Commit file env.txt"
04
Merge dev → stage
git checkout stage  →  git merge dev
05
Actualizare repository remote
git push origin stage
06
Evaluare rezultat — II. Branch
env.txt apare în branch-ul stage pe GitHub
III. Branch

Redenumire dev → develop

01
Redenumire locală branch dev → develop
git branch -m dev develop
02
Actualizare repository remote
git push origin develop  →  git push origin --delete dev
03
Evaluare rezultat — III. Branch
Branch dev dispare pe GitHub; apare develop în locul lui
